# AI-OS v1.2 Roadmap

Prioritized improvements justified by operational value. **Not implemented** — planning document only.

Grounded in production audit (2026-07) and personal configuration work. No new architectural layers.

---

## Immediate (next 1–2 weeks)

Operational fixes that improve daily use without changing system design.

| # | Item | Problem | Value |
|---|------|---------|-------|
| 1 | **Refresh stale documentation** | `README.md`, `CHANGELOG.md`, `tests/README.md`, `system-overview.md` still describe "foundation phase" | Prevents wrong onboarding and runbooks |
| 2 | **Lightweight `ai-os status`** | Full integrity scan on every status call | Faster daily CLI on growing corpora |
| 3 | **Google OAuth refresh helper** | `GOOGLE_ACCESS_TOKEN` expires hourly | Gmail/Calendar/Drive usable without manual token paste |
| 4 | **Silent YAML load failures** | Invalid workflow/automation YAML skipped without error | Scheduled automations can disappear silently |
| 5 | **Align `doctor` vs `maintenance doctor` defaults** | Different `--repair` defaults confuse operators | Predictable maintenance behavior |
| 6 | **Fix flaky chat import test** | `test_import_chats_json` fails in full suite | Reliable CI |

**Exit criteria:** Docs match v1.1; `status` < 2s on typical corpus; Google token refresh documented or scripted; YAML errors surfaced in CLI.

---

## Next Month (weeks 3–6)

Performance and consistency improvements for growing personal knowledge base.

| # | Item | Problem | Value |
|---|------|---------|-------|
| 7 | **Single-pass memory retrieval** | `retrieve_for_task()` scans all memories 4× per workflow | Scales as `memory/` grows |
| 8 | **Cache HybridSearch instances** | New Chroma client + BM25 reload per search call | Faster search and skill execution |
| 9 | **Dashboard without live API calls** | `dashboard`/`diagnostics` hit all provider APIs | Works offline; no rate limits |
| 10 | **Wire or remove `index.jsonl` sidecars** | Written but never read in memory/decision/automation stores | Less disk churn; clearer listing |
| 11 | **Unify skill/tool knowledge glue** | Duplicate logic in `capabilities/helpers.py` and `agent/tools/builtin.py` | Consistent retrieval/decision behavior |
| 12 | **Restore dry-run verification** | Backup verify exists; restore not tested | Confidence in disaster recovery |
| 13 | **Personal import scripts** | Manual `ai-os import` commands per category | `scripts/import-personal.sh` wrapping import plan phases |
| 14 | **pyproject version alignment** | Package version `0.1.0` vs tags `v1.1.x` | Support and debugging clarity |

**Exit criteria:** Workflow task with 100+ memories < 500ms context build; dashboard instant offline; backup + restore verified in `doctor --full`.

---

## Future (3+ months)

Improvements deferred until daily usage surfaces concrete need.

| # | Item | Trigger to prioritize |
|---|------|----------------------|
| 15 | **Automation daemon** | User runs `automation tick` manually and wants background scheduling |
| 16 | **Secondary provider implementations** | Notion, Slack, Discord needed for workflows |
| 17 | **Memory search index** | Memory count > 500; linear scan becomes noticeable |
| 18 | **Integrity check caching** | Corpus > 10k chunks; doctor takes > 30s |
| 19 | **Encrypted memory at rest** | `ENCRYPTION_KEY` in `.env.example` used for sensitive legal docs |
| 20 | **Web UI** | `FEATURE_WEB_UI=true` justified by daily non-CLI use |
| 21 | **Shared `utc_now()` utility** | Timestamp drift causes cross-layer debugging pain |
| 22 | **CLI smoke test suite** | Regression in `dashboard`, `doctor --full`, workflow run |

## Explicitly out of scope for v1.2

Items considered and rejected — no operational justification today:

- New provider types (beyond existing 16)
- New capability/skills beyond fixing existing 16
- Knowledge Engine architecture changes
- Decision Engine strategy additions
- Multi-user or deployment infrastructure
- Database backend migration (SQLite/file is sufficient for personal use)
- Real-time collaboration features
